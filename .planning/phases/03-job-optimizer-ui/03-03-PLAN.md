  <name>Create useJobAnalyzer hook for JD-only analysis</name>
  <files>src/features/job-optimizer/hooks/useJobAnalyzer.ts</files>
  <action>
Create custom hook for analyzing job description text independently of resume data.

Hook requirements:
- Input: jobDescription (string)
- Return type: { overview: OverviewData, requirements: RequirementsData, keywords: KeywordsData, insights: InsightsData, isAnalyzing: boolean }
- Use useMemo with 500ms debounce for analysis (avoid re-computing on every keystroke)
- No AI calls in MVP—rule-based analysis only

Analysis functions:
1. extractOverview(text: string): Extract role title (first line or "Job Title:" pattern), seniority level (keywords: senior, junior, lead, principal), company (if mentioned), location (remote, hybrid, onsite patterns)
2. extractRequirements(text: string): Split into required vs preferred (sections with "required", "must have" vs "preferred", "nice to have"), return arrays of bullet points
3. extractKeywords(text: string): Extract technical terms, tools, skills—remove common words (the, and, or, etc.), return array of { keyword: string, frequency: number } sorted by frequency
4. extractInsights(text: string): Simple heuristics—detect if role is IC vs manager (keywords: team, direct reports, leadership), tech stack mentioned (languages, frameworks), experience years (regex: \d+ years)

Implementation:
- Use lodash debounce for input text
- Return isAnalyzing: true during debounce period
- Export named: export function useJobAnalyzer(jobDescription: string)
- All analysis is synchronous and rule-based (no async operations)
